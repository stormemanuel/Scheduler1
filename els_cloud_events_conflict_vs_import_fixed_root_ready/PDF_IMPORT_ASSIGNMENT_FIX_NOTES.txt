ELS Events PDF Import Assignment Fix

This build keeps the PDF parser fixes from the studied ADA/AAPA/KIDNEY/VS EXCHANGE PDFs and fixes the import save path that prevented matched crew from being inserted as assignments.

Root cause fixed:
- The PDF parser produced sub-call times as HH:MM.
- Supabase TIME columns are returned as HH:MM:SS after insert.
- The importer looked up created sub-calls using a key that included time, so 08:00 did not match 08:00:00.
- Result: sub-calls were created correctly, crew_needed was correct, but matched crew were not attached to assignments.

Code changed:
- app/api/events/import/route.ts
  - Added normalizeTimeKey().
  - makeSubCallKey() now normalizes HH:MM and HH:MM:SS to the same key.
  - Assignment insertion now re-computes the normalized key instead of using the preview-only raw key.
  - Import success message now reports how many crew were actually assigned.
  - Added a guard error if the importer matched crew but cannot attach them to sub-calls.

Database data preserved:
- No schema reset.
- No contact/rate/crew changes.
- Existing New Orleans/Nashville/Atlanta pools are preserved.
