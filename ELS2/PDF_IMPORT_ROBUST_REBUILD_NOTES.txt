PDF Import Robust Rebuild Notes

This build changes the PDF importer from one extraction path to a scored multi-parser flow:

1. Server reads each PDF in two extraction variants:
   - default PDF text order
   - visual x/y reconstructed row order

2. Importer parses and scores both variants, then chooses the cleanest result.

3. Handles known ELS crew list variations:
   - ADA-style numeric dates and booth headers across page breaks
   - AAPA-style multiple booth headers before multiple tables
   - VS Exchange split weekday/date/time/role rows and OPTOS booth continuation
   - KIDNEY-style month dates and hyphen time ranges
   - PAYROLL-style short numeric date ranges and compact single-booth tables
   - PO columns and PO numbers embedded in booth headers
   - split row details across pages
   - OCR/PDF artifacts and spaced-out text extraction
   - On call / Available rows are ignored

4. Keeps existing features:
   - role aliases and payout aliases
   - crew matching and assignment creation
   - schedule conflict blocking
   - day view / booth view selector
   - day and booth crew-list exports
   - inline edit tools

5. Adds clearer import request errors when the serverless route crashes or times out.
