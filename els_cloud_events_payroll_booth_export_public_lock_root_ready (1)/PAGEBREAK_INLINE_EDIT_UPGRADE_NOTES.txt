ELS Events Upgrade - PDF Page Break + Inline Edit

What changed:
1. PDF import now renders text page-by-page in visual top-to-bottom / left-to-right order before parsing.
   This is intended to keep rows that continue onto the next PDF page under the correct booth/area.

2. PDF parser now queues booth/area headers and activates them by table block.
   This helps with real labor call lists where:
   - booth headers are stacked before multiple tables,
   - rows continue onto the next PDF page,
   - the next booth header appears in extracted text before continuation rows.

3. Events edit forms now appear inline:
   - Edit Event opens under the event summary.
   - Edit Day opens under that labor day.
   - Add Sub-Call opens under that labor day.
   - Edit Sub-Call opens directly under that sub-call card.

Preserved from prior builds:
- PDF / CSV import feature
- crew matching and assignment on import
- role alias payout fixes
- day view / booth view selector
- booth-style export and day-separated export

Deployment:
- package.json is at the root of this zip.
- Vercel Root Directory should remain blank.
